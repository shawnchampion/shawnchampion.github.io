Install Hexo-v3.2.2 and simple usage

Posted on 2017-01-26

This tutorial will show you how to install node and hexo on your ubuntu and deploy a blog on github.

Install Node.js and npm

Download the latest package on the official website

Click here to download node-v6.9.4-linux-x64.tar.xz or you can visit download page for other suitalbel version.

Extract the package and install

Enter the directory you download it. Extract package and move to /opt directory to install.

Add softlinks for node and npm (package manager for JavaScript)

Now you can run node -v and npm -v to check it. You can find

v6.9.4
3.10.10

on your terminal if installed correctly

Install Hexo

Use npm to install hexo and add a softlink.

At this time, when you type in hexo -v, you can see

hexo: 3.2.2
hexo-cli: 1.0.2
os: Linux 4.4.0-59-generic linux x64
http_parser: 2.7.0
node: 6.9.4
v8: 5.1.281.89
uv: 1.9.1
zlib: 1.2.8
ares: 1.10.1-DEV
icu: 57.1
modules: 48
openssl: 1.0.2j

Start Build your blog with hexo

Init your blog

The codes above play these roles:

  1. init a folder to manage your blog, the whole content of your blog will be here
  2. enter blog directory
  3. install related components(I guess :D )
  4. start local server

After all these, type localhost:4000 in web browser’s address bar to visit the welcome page!

Qucik start

  1. create a new article in your blog, edit it in the directory /source/_posts(Markdown supported).
  2. start local server for a preview
  3. according to previous modifies, generate static files in local space.
  4. deploy your website

However, if you try to deploy it right now, there will be several mistakes.

Before deployment

Take github for example, open the file _config.yml in your blog directory, find the label # deploment and edit it like the following:

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: your_repository_address
  branch: master

and load git deployer package.

The last step, make sure git installed on your ubuntu,run git version to check.If not,qucikly install it by sudo apt install git

If everything well-settled, now you can deploy your blog!

During the deployment,your github username and password would be asked.Remember hexo generate before every deployment,or nothing will change.

PS:Once after hexo deploy, fatal: Not a git repository (or any of the parent directories) happened, delete the file .deploy_git and try it again will help.
